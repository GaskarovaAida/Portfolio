# A/B-тест SollmaFin — анализ нового онбординга

## Описание

Fintech-сервис SollmaFin обновил онбординг инвестиционного приложения: новый флоу содержит подробную информацию о рисках активов. Цель — проверить, улучшает ли новый онбординг вовлечённость и монетизацию пользователей.

---

## Задачи

1. Предобработка и EDA исторических данных: воронка событий, динамика привлечения по странам
2. Анализ влияния уровня риска актива на конверсию во второй депозит
3. Расчёт базовой монетизации (ARPU) на исторических данных
4. Проверка корректности A/B-теста: распределение по группам (хи-квадрат), пересечения
5. Сравнение воронок контрольной и тестовой групп
6. Оценка метрик: ARPU (t-тест Уэлча), конверсии (z-тест пропорций), ARPPU
7. Bootstrap-анализ суммы депозитов платящих пользователей (25/50/75 перцентили)

---

## Инструменты

- **Python** — основной язык анализа
- **pandas, NumPy** — обработка данных
- **SciPy** — t-тест Уэлча, z-тест пропорций, хи-квадрат
- **Bootstrap** — анализ перцентилей суммы депозитов
- **Matplotlib, Seaborn** — визуализация воронок и распределений

---

## Результаты

- ARPU не улучшился статистически значимо (p ≈ 0,17)
- Конверсия из первого депозита во второй статистически значимо выросла в тестовой группе
- ARPPU (средний депозит платящего пользователя) вырос в тестовой группе
- Bootstrap: мелкие вкладчики стали инвестировать меньше, крупные — больше
- Фича работает не для всех: смещает распределение вкладов, а не поднимает среднее
